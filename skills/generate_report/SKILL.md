# generate_report

## Purpose

Create a reproducible research report from experiment context, metrics, diagnostics, and artifacts.

## Expected Inputs

- Experiment ID
- Config path
- Metrics
- Risk diagnostics
- Artifact manifest

## Expected Outputs

- Markdown or HTML report
- Artifact references
- Assumptions and limitations
- Next-step recommendations

## Workflow

1. Load experiment metadata and artifact manifest.
2. Summarize setup, assumptions, and validation status.
3. Present metrics and diagnostics with traceable sources.
4. State limitations before recommendations.
